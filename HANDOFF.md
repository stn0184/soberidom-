## 1. Что за проект (в двух абзацах)

**СобериДом** — веб-сервис «IKEA-инструкция для постройки каркасного дома своими руками». Человек без опыта стройки вводит параметры (тип дома, площадь, регион, отопление), получает подбор проектов с рекомендацией фундамента по местности, бесплатно смотрит 3D и предварительную смету, а после оплаты получает интерактивный конструктор: пошаговая сборка «возьми доску, отметь красным маркером, отрежь 2440 мм, прибей сюда», раскрой, живая смета, фин-отчёт план/факт, волны доставки, скрипты переговоров с подрядчиками и AI-«Добрый прораб».

Ниша пустая: конкуренты — либо сухие инженерные калькуляторы (kalk.pro) для тех, кто уже разбирается, либо статичные PDF-библиотеки (karkasdom.info), либо лидогенераторы стройкомпаний. Никто не совмещает подбор по местности + конструктор «для того, кто не держал молоток» + AI-сопровождение. География: старт СНГ (RU/KZ/BY), цель — мир.

## 2. Методология

Проект строится по курсу «AI-Архитектор», Spec-First Pipeline:
```
Идея → PROJECT_IDEA.md → SPEC.md → CLAUDE.md → сборка через Claude Code по этапам → Git/деплой
```
Роли: пользователь (владелец продукта) — принимает решения, валидирует; ведущий-ассистент — структурирует, генерирует документы, ведёт по шагам; Claude Code — собирает код строго по SPEC.md.

**SPEC.md — единственный источник истины.** При любом противоречии между кодом, CLAUDE.md и SPEC.md — побеждает SPEC.md. Новые идеи не ломают текущий этап: идут в бэклог (раздел 7) и добавляются в SPEC через правки.

## 8. Открытые вопросы / технический долг

- **Email-шаблоны Supabase:** письма подтверждения регистрации по умолчанию могут не вести на /auth/confirm. Починка — в дашборде Supabase (Auth → Email Templates), шаблон должен использовать token_hash и вести на /auth/confirm. Сделать перед живым тестом регистрации (не блокирует этап 2).
- **MCP Supabase на Windows:** подключается «наполовину» (Claude Code ходит в базу через Management API, не через нативные MCP-инструменты). Не блокирует. Если чинить — заменить в .mcp.json "command":"cmd" на прямой npx.cmd.
- **Токен Supabase:** личный access token лежит в открытом виде в .mcp.json (файл в .gitignore). На будущее — вынести в переменную окружения. Токен, который был случайно показан в чате ранее, при выходе в продакшн — отозвать и пересоздать.
- **Название репозитория** `soberidom-` с лишним дефисом (косметика, можно переименовать в GitHub Settings).
